Resizable does not properly calculate width / height when box-sizing is different

Reported by: kenaniah Owned by: kenaniah
Priority: minor Milestone: 2.0.0
Component: ui.resizable Version: 1.8.23
Keywords: Cc:
Blocked by: Blocking:
Description

All elements when resized are currently (as of 1.8.23) calculated using .width() and .height(), but are saved using .css(). The CSS interpretation of the width / height properties is subject to the box-sizing property.

As a result of this approach, elements that have a box model other than content-box will "jump" upon the initialization of a resize operation due to the miscalculation of the initial width and height.

An example of this behavior can be found when resizing the dialog on the demo page (http://jqueryui.com/demos/dialog/#default) in Chrome.
