droppable("disable") behaves differently in Google Chrome than other browsers

Reported by: sgraffite Owned by: sgraffite
Priority: minor Milestone: 2.0.0
Component: ui.droppable Version: 1.7.1
Keywords: droppable chrome Cc:
Blocked by: Blocking:
Description

When you have a droppable that is enabled, and drop an item in it which disables that container, you can no longer drag that item out of the container in Google Chrome. You are able to remove that item in Firefox 2, Opera 9, and IE 6/7.

Repro steps:

1. Start up Firefox, Opera, or IE.

2. Go to http://www.geocities.com/filidhsacrificer/test.html

3. Drag the X icon from the top droppable to the smaller bottom droppable. The bottom container will now be disabled, and no longer accept draggables.

4. Drag the X icon back to the original container.

5. Start up Chrome.

6. Go to http://www.geocities.com/filidhsacrificer/test.html

7. Drag the X icon from the top droppable to the smaller bottom droppable. The bottom container will now be disabled, and no longer accept draggables.

8. Attempt to drag the X icon back to the original container. It won't give you the pointer cursor and won't respond to click and drag (unless you placed the icon in the droppable in such a way that part of it sticking out over the side, then you can still drag it out using that part).
