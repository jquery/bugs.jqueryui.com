Unicode Support in Trac

Trac encodes all text using UTF-8, including text in tickets and wiki pages. Internal processing of text uses true Unicode representations. As such, it supports the most commonly used character encodings.

If the default encoding of your source code repository is not UTF-8, you can specify it in [TracIni#trac-default_charset-option "[trac] default_charset"]:

default_charset = gbk

Ensure that your database stores its data in UTF-8, otherwise results may be unpredictable.

To convert your database to UTF-8, the easiest way is to create a dump of the database, convert it into UTF-8, for example using iconv, and then re-import it.
